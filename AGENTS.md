# AGENTS.md – Shuriza Store

Laravel 12 digital-products e-commerce store ("Shuriza Store Kediri"). Blade-based frontend, SQLite database (dev), Indonesian-language UI.

---

## Architecture Overview

```
routes/web.php          → all HTTP routes (frontend + admin + auth)
app/Http/Controllers/   → frontend controllers (Cart, Order, Product, Home, Profile)
app/Http/Controllers/Admin/ → admin-only controllers (same resource names, aliased in routes)
app/Models/             → Eloquent models (User, Product, Category, Order, OrderItem, CartItem)
resources/views/        → Blade views; layouts/app.blade.php (public) + layouts/admin.blade.php
bootstrap/app.php       → middleware alias: `admin` → AdminMiddleware
```

**Role system:** `users.role` column is `'admin'` or `'user'`. Use `$user->isAdmin()` / `$user->isUser()`. Admin routes use `->middleware(['auth', 'admin'])`.

**Cart duality:** `CartItem` rows are keyed by either `user_id` (authenticated) or `session_id` (guest). `CartController` handles both transparently via `cartCondition()` / `getCartQuery()`.

**Order flow:** Checkout does _not_ require auth; a `user_id` is nullable. After `Order::store()`, the user is redirected to `order.success` which shows a WhatsApp deep-link (`wa.me/`) built by `Order::getWhatsAppUrl()`. Admin phone comes from `config('app.whatsapp_number', '6281234567890')`.

## Developer Workflows

```bash
# First-time setup
composer run setup          # install deps, copy .env, key:generate, migrate, npm install + build

# Daily development (starts server + queue + pail + Vite concurrently)
composer run dev

# Run tests
composer run test           # clears config cache then runs PHPUnit

# Seed database (creates admin + test user + categories + products)
php artisan db:seed
```

Seed credentials: `admin@shurizastore.com` / `admin123` and `user@example.com` / `password`.

After adding images, run `php artisan storage:link` once to expose `storage/app/public` as `public/storage`.

## Key Conventions

**Slug auto-generation:** Both `Product` and `Category` auto-generate `slug` from `name` on create (and on update if `name` changes but `slug` doesn't). Product uses `slug` as route key (`getRouteKeyName()`), so routes are always `/produk/{slug}`.

**Prices:** Stored as plain integers (IDR, no decimals). Format with `$product->formatted_price` accessor → `"Rp 100.000"`. Cast `price` and `original_price` as `'integer'` in `$casts`.

**Model scopes to use consistently:**
- `Product::active()` – `is_active = true`
- `Product::inStock()` – `stock > 0`
- `Product::sorted($sort)` – handles `price-asc`, `price-desc`, `name-asc`, `popular`, `default`
- `Category::active()->ordered()` – active categories by `sort_order`

**Order numbers:** Format `SHR-YYMMDD-XXXXX` (uppercase) — generated by `Order::generateOrderNumber()`.

**Order statuses:** `pending` → `processing` → `completed` / `cancelled`. Label and badge-color accessors live on the `Order` model (`status_label`, `status_badge`, `status_color`).

## Views & Styling

Two layouts — never mix them:
- `layouts.app` → public-facing pages (dark theme)
- `layouts.admin` → admin area

Dark-theme CSS variables (defined in both layouts' `<style>` blocks):
```css
--primary: #6c63ff;  --secondary: #ff6584;  --accent: #43e97b;
--dark: #0f0e17;     --dark2: #1a1926;       --card-bg: #1e1d2e;
```

Pages extend a layout and push page-specific CSS: `@extends('layouts.app')` → `@push('styles')`. Icons use **Font Awesome 6.5 CDN** (`fas fa-*` classes). Interactivity uses **Alpine.js** (`x-data`, `x-show`, etc.) loaded via Vite.

Blade components (from Breeze) live in `resources/views/components/` — prefer `<x-primary-button>`, `<x-input-label>`, `<x-text-input>`, etc. for forms.

## Integration Points

| Concern | Location |
|---|---|
| WhatsApp order notification | `Order::buildWhatsAppMessage()` / `getWhatsAppUrl()` — configure phone in `config/app.php` as `whatsapp_number` |
| Product images | Stored at `storage/app/public/{path}`; accessed via `$product->image_url` accessor |
| Auth | Laravel Breeze (`routes/auth.php`); session-based |
| Frontend assets | Vite (`vite.config.js`); entry: `resources/css/app.css` + `resources/js/app.js` |
| Admin AJAX (bulk, toggle, search) | Uses standard form POST with `@csrf`; JSON responses for `expectsJson()` requests |
